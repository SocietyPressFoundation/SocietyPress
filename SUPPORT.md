# Getting Help with SocietyPress

SocietyPress is written for people who volunteer their time to run
genealogical and historical societies, not for software developers. This
page lists the right places to go when you get stuck — in rough order
of what to try first.

## 1. Check the documentation

Most questions are answered in the documentation.

- **End-user guides** — [getsocietypress.org/docs](https://getsocietypress.org/docs)
  cover installing, setup, and every module from a volunteer's
  point of view.
- **Feature reference** — [`docs/FEATURES.md`](docs/FEATURES.md) in this
  repository is the authoritative list of what every module does.
- **Frequently asked questions** — [getsocietypress.org/faq](https://getsocietypress.org/faq)

## 2. Ask on the community forums

For "how do I …" questions, workflow discussions, and anything that might
benefit another society administrator:

**[getsocietypress.org/community](https://getsocietypress.org/community)**

The forums are the best place for general help. Discussions are archived
and searchable, so an answer posted today helps the next administrator
who runs into the same thing next year.

## 3. File a bug report

If you believe SocietyPress is broken — it's doing something it shouldn't,
or refusing to do something it should — file a bug report:

**[github.com/SocietyPressFoundation/SocietyPress/issues](https://github.com/SocietyPressFoundation/SocietyPress/issues)**

Please read [`CONTRIBUTING.md`](CONTRIBUTING.md) first for what makes a
useful report.

## 4. Send us feedback

For suggestions, comments, frustrations, or anything that isn't a specific
bug or how-to question:

**[getsocietypress.org/feedback](https://getsocietypress.org/feedback)**

## 5. Report a security issue (do this privately)

Do not file security issues publicly. See [`SECURITY.md`](SECURITY.md)
for the reporting process.

## What to include when asking for help

The more of this you can tell us, the faster we can help:

1. **What you were trying to do** (the goal, not just the action)
2. **What you expected** to happen
3. **What actually happened**, word for word if there's an error message
4. **Your SocietyPress version** — visible at the bottom of any admin
   page, or under *Dashboard → About*
5. **Your WordPress version** and web host, if you know them
6. **Screenshots** of the issue, if visual

It's always okay to say "I don't know what's going on, but something's
wrong." We'd rather hear from you early than have you struggle in silence.

## What not to send

- **Passwords, API keys, or database credentials** — we never need these
  and posting them publicly is a security risk
- **Your members' personal data** — export a sample only if absolutely
  necessary, and never attach a full member export to a public issue
- **Your WordPress admin URL or login page** — we don't need it

## Response times

SocietyPress is maintained by one person. We do our best to reply
promptly, but there's no SLA and no paid support tier. Security issues
are prioritized; everything else is best-effort.

If you need guaranteed support response times, you're welcome to hire a
WordPress consultant of your choice — the code is open source and the
documentation is thorough, so any competent WordPress developer can help
you run it.
