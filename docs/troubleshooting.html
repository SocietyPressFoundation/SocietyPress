Troubleshooting

Having trouble? This page covers common problems and their solutions. If you don't find your answer here, see the "Getting Help" section at the bottom.

Login & Access Problems

I can't log in to WordPress

Forgot your password?

  1. Go to yoursite.com/wp-login.php
  2. Click "Lost your password?"
  3. Enter your email address
  4. Check your email for a reset link

Password not working?

  • Make sure Caps Lock is off
  • Try copying and pasting the password to avoid typos
  • Clear your browser cache and cookies

Still can't get in?

Contact your hosting provider. They can reset your WordPress password through the database.

The site says I don't have permission

This usually means your WordPress user account doesn't have administrator privileges.

Solution:

  1. Have another administrator log in
  2. Go to Users → All Users
  3. Find your account and click "Edit"
  4. Change the "Role" to "Administrator"
  5. Save the user
I see "SocietyPress plugin required" notice

The SocietyPress theme is active but the plugin isn't installed or activated.

Solution:

  1. Go to Plugins → Installed Plugins
  2. Find "SocietyPress" in the list
  3. Click "Activate"

If you don't see SocietyPress in the plugins list, you need to install it first. See the Installation Guide.

Member Problems

Members aren't showing up in the list

Check your filters:

  1. Go to SocietyPress → Members
  2. Look at the filter dropdowns at the top
  3. Make sure "All Statuses" and "All Tiers" are selected
  4. Clear any search terms in the search box

Did the import fail?

If you recently imported members, check the import results. Some records may have been skipped due to missing required fields.

I imported members but some are missing

The import may have skipped some records. Check the import results screen for details on skipped records and why.

Common reasons for skipped records:

  • Missing required fields (first name, last name, or email)
  • Duplicate email address (if "skip duplicates" was selected)
  • Invalid date format
  • Empty rows in the spreadsheet

Fix the issues in your spreadsheet and re-import the missing records.

Member expiration dates are wrong

Check the expiration model:

  1. Go to SocietyPress → Settings
  2. Find the "Expiration Model" setting
  3. Choose the correct model:
    • Calendar Year - Everyone expires Dec 31
    • Rolling - Expires X months after joining

Note: Changing the model doesn't automatically update existing members' expiration dates. You may need to manually adjust them.

Event Problems

Events aren't showing on the homepage

Check that events exist and are published:

  1. Go to SocietyPress → Calendar
  2. Make sure you have upcoming events (date in the future)
  3. Make sure events are "Published" not "Draft"

Check the homepage template:

The SocietyPress theme shows events on the homepage. If you're using a different theme or a custom homepage, events may not appear.

Members can't register for events

Check registration is enabled:

  1. Edit the event
  2. Find the "Registration" section
  3. Make sure "Registration required" is checked

Check if slots have capacity:

  • If using time slots, verify they have available capacity
  • If all slots are full, new registrations go to the waitlist

Check if member is logged in:

Members must be logged into their WordPress account to register. Make sure they have an account linked to their member record.

Recurring events aren't generating

Recurring events in SocietyPress are shown dynamically, not created as separate posts. They appear on the calendar based on the recurrence pattern.

If recurring events aren't showing, verify the recurrence settings on the event and make sure the event is published.

Email Problems

Emails aren't being sent

Check if you're in development mode:

When WordPress debug mode is on (WP_DEBUG = true) or you're on localhost, SocietyPress blocks outgoing emails but logs them for review.

  1. Check your wp-config.php
  2. Set WP_DEBUG to false for production

Check the Email Log:

SocietyPress logs all outgoing emails. Look for the Email Log in the admin to see what was sent (or blocked).

Email deliverability issues:

Many hosting providers have poor email delivery. Consider using an SMTP plugin (like "WP Mail SMTP") to send through a proper email service like Gmail, SendGrid, or Amazon SES.

Emails are going to spam

This is usually a server configuration issue, not a SocietyPress problem.

Solutions:

  • Set up SPF records for your domain
  • Set up DKIM signing
  • Use an SMTP plugin to send through a reputable email service
  • Make sure the "From" email matches your domain

Contact your hosting provider for help with email configuration.

Display Problems

The site looks broken or unstyled

Check the theme is active:

  1. Go to Appearance → Themes
  2. Make sure the SocietyPress theme is active

Clear caches:

  • Clear your browser cache
  • If using a caching plugin, clear its cache
  • If your host has server-side caching, clear that too

Check for plugin conflicts:

Temporarily deactivate other plugins to see if one is causing the problem. Reactivate them one by one to find the culprit.

Menu items aren't showing

Check menu assignments:

  1. Go to Appearance → Menus
  2. Select your menu
  3. Under "Menu Settings," verify it's assigned to the right location

Smart menu filtering:

Remember that SocietyPress automatically hides certain menu items:

  • "Join" and "Register" links hide for logged-in users
  • "Directory" and "Portal" links hide for logged-out visitors

This is normal behavior. Log in/out to see how the menu changes.

Performance Problems

The site is running slowly

Common causes and solutions:

1. Hosting:

  • Cheap shared hosting often has poor performance
  • Consider upgrading to better hosting if speed is important

2. Caching:

  • Install a caching plugin like WP Super Cache or W3 Total Cache
  • This dramatically speeds up page loads for visitors

3. Images:

  • Large, unoptimized images slow down pages
  • Use an image optimization plugin
  • Resize images before uploading

4. Too many plugins:

  • Each plugin adds overhead
  • Deactivate plugins you don't need
